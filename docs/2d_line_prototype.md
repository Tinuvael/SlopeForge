# Прототип 2D-линий Datamine

Это временный изолированный прототип для проверки ручного выбора сегментов линий на 2D-плане. Он не меняет основную модель SlopeForge, не создаёт таблицы PostgreSQL и не использует PostGIS.

## CSV Datamine

Поддерживается импорт CSV с колонками Datamine:

- `XP`, `YP`, `ZP` — координаты точки;
- `PTN` — идентификатор исходной линии;
- `PVALUE` — дополнительное значение точки, необязательное;
- `TYPE` — исходный тип линии, необязательное.

Регистр названий не важен. Минимально нужны `XP`, `YP`, `ZP`, `PTN`. Если они не найдены автоматически, прототип показывает диалог сопоставления колонок.

Строки с одинаковым `PTN` собираются в одну полилинию. Порядок точек сохраняется по порядку строк CSV. Координаты читаются без специального округления.

## Рабочий сценарий

1. Открыть кнопку **2D Plan Prototype** в главном окне.
2. Импортировать CSV.
3. При необходимости отфильтровать линии по отметке, `TYPE`, назначенному типу или `PTN`.
4. Выбрать линию кликом на плане.
5. Включить режим **Выбрать сегмент**.
6. Кликнуть начало и конец сегмента на одной линии.
7. Задать роль: `upper_boundary`, `lower_boundary` или `intermediate_assessment`.
8. Создать черновик уступа `U-001` из верхнего и нижнего сегментов.
9. Добавить промежуточные сегменты вручную.
10. Сохранить прототип в JSON и загрузить его при следующем запуске.

## Выбор сегмента

Клик не обязан попадать точно в вершину. Координата курсора проецируется на ближайший отрезок выбранной полилинии. Прототип хранит положение начала и конца вдоль линии и отдельно сохраняет извлечённые точки сегмента, чтобы черновик оставался стабильным после повторного импорта изменённой линии.

## Хранение JSON

По умолчанию состояние сохраняется в `SlopeForge/prototype/line_segment_prototype.json` внутри каталога `QStandardPaths.AppDataLocation`. Если Qt не вернул путь, используется fallback `~/.config/SlopeForge/prototype/line_segment_prototype.json`.

В JSON сохраняются импортированные линии, исходный `source_type`, пользовательский `assigned_type`, выбранные сегменты и черновики уступов.

## Ограничения

- Данные прототипа не являются производственными.
- PostgreSQL, Alembic и SQLAlchemy не используются.
- Нет расчёта RMS, перебора, недобора и аналитических ячеек.
- Нет автоматического связывания с взрывными событиями.
- Нет 3D-каркасов, PyVista, VTK, OpenGL и полноценного GIS.
- Принадлежность промежуточных сегментов уступу задаётся только вручную.
