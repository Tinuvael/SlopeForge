# Прототип 2D-плана Datamine

Это изолированный инструмент проверки геометрии и ручного создания сегментов уступов. Он не использует PostgreSQL, Alembic, PostGIS или производственные сущности SlopeForge. Данные прототипа не считаются производственными.

## CSV и Dataset

Поддерживаются CSV Datamine с `X`/`XP`, `Y`/`YP`, `Z`/`ZP`, `SID` и, при наличии, `PTN`, `PID`, `TYPE`, `PVALUE`. Каждый импорт создаёт Dataset (`D-001`, `D-002` и т. д.). Сегмент хранит `dataset_id`, SID и frozen geometry, поэтому новый CSV не меняет ранее созданные уступы.

## Компоновка редактора

- **Слева** — текущий проект, активный Dataset, список уступов и единственная кнопка **«+ Добавить уступ»**.
- **В центре** — 2D-план с импортом, сохранением, Fit to extent, рабочим горизонтом, сеткой и координатами.
- **Справа** — только состав выбранного уступа: верхняя/нижняя граница и промежуточные сегменты.
- **Снизу** — компактная workflow-панель. Она скрыта в обычном режиме и появляется только при создании либо редактировании.

Функция «Граница карьера», её кнопки, стили и контекстные действия удалены. Поле `semantic_role` остаётся только для безопасной загрузки старых JSON и не меняет отображение линий.

## Работа с уступом

Нажмите **«+ Добавить уступ»**, выберите нижнюю линию и задайте A/B. После подтверждения нижняя граница остаётся на плане, пока выбирается верхняя линия. После создания уступ появляется слева, становится выбранным, а workflow-панель скрывается.

Для выбранного уступа правая карточка показывает верхнюю и нижнюю границы, Dataset и список промежуточных сегментов. Добавление промежуточного сегмента выполняется только вручную; во время выбора остаются видимыми обе границы и уже добавленные промежуточные сегменты.

Верхняя граница по умолчанию является первой промежуточной оценкой. Это ссылка на тот же `upper_segment_id`, не второй сегмент с копией геометрии. Такое участие можно отключить в карточке уступа.

## Выбор и горячие клавиши

На этапе выбора линии клик формирует список ближайших кандидатов. Стрелки `←`/`↑` и `→`/`↓` переключают только кандидатов; `Enter` закрепляет нужную линию. В этапах установки A/B стрелки ничего не меняют. Обычное колесо выполняет zoom; стрелки и `Ctrl+колесо` не переключают горизонты.

`Enter` подтверждает текущий шаг, `Backspace` возвращает на предыдущий, `Escape` отменяет незавершённый шаг или очищает выделение в обычном режиме. `Delete` работает только с явно выбранным объектом и требует подтверждения там, где объект удаляется.

## Отображение

На плане одновременно подсвечивается только один выбранный уступ. При выборе другого уступа графика предыдущего полностью снимается. После снятия выделения остаются только исходные Datamine-линии.

Маркеры A и B равноправны и перетаскиваются только вдоль активной исходной линии. Во время drag обновляется preview, а frozen geometry меняется только после сохранения.

## Хранение и ограничения

JSON сохраняется в `SlopeForge/prototype/line_segment_prototype.json` внутри `QStandardPaths.AppDataLocation`, с fallback `~/.config/SlopeForge/prototype/line_segment_prototype.json`.

Нет 3D-каркасов, RMS, перебора/недобора, автоматической перепривязки к новому Dataset, PostGIS, PyVista, VTK, OpenGL и multi-line selection.
